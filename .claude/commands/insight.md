# /insight — 질문 → 인사이트 글 (직접 경로)

질문 / 생각: **$ARGUMENTS**

원문 URL 이 없는 글이다. 사용자의 질문·주장이 논지가 되고, 이 세션이 웹 리서치로
근거를 모아 의견 글을 완성해 Supabase `posts` 에 `draft` 로 넣는다.

## 절차

1. **규약 숙지** — `INSIGHT.md`(이 장르의 규약)와 `POSTING.md`(공통 규약 — 2-0 제목,
   2 말투·골격, 2-1 강조, 2-2 시각자료, **2-3 자동화 티 방지**, 3 출처)를 읽는다.
   [[korean-writing-style]] 도 지킨다.
2. **논지 세우기** — 입력에서 **질문**과 **사용자의 주장**을 분리한다.
   - 주장이 함께 왔으면 그것이 글의 논지다.
   - 질문만 왔으면 이 세션이 관점을 세우되, 글을 쓰기 전에 "이런 논지로 쓰겠다"를
     한 줄로 사용자에게 확인한다.
3. **직전 초안 점검** — `drafts/` 의 최근 파일 1~2개를 훑어 길이·헤드라인 개수·도입
   형태·시각자료 조합을 확인하고, 최소 2~3가지를 의도적으로 다르게 잡는다(2-3절).
4. **리서치** — `WebSearch`·`WebFetch` 로 핵심 근거 **5~10건**을 찾아 읽는다.
   - 논지를 받치는 근거와 **반대 시각의 근거를 모두** 찾는다(INSIGHT.md 3절).
   - 1차 자료(공식 블로그·논문·공식 문서·발표·통계) 우선. 출처 URL·발행일을 메모해 둔다.
   - 조사 결과가 논지와 어긋나면 사실을 우선한다 — 사용자에게 알리고 논지를 수정하거나
     글의 전환점으로 정직하게 쓴다.
5. **초안 작성** — INSIGHT.md 골격(질문 → 내 관점 → 근거 → 반론 → 결론)으로 쓴다.
   - 화자는 글쓴이 1인칭. 사실은 출처로 단정, 해석은 해석임이 드러나게.
   - 본문 인용은 인라인 링크, 말미에 `## 참고 자료` 로 전체 출처를 모은다. 각주 ✗.
   - 제목은 질문 그대로 ✗ — 결론·가장 강한 한 가지로 후킹(POSTING.md 2-0절).
6. **문체 퇴고 — 번역투 제거 전용 패스 (필수).** 초안을 다 쓴 뒤, 사실·구조·링크는
   그대로 두고 **문장만** 처음부터 다시 읽으며 한국어 어순으로 고쳐 쓴다.
   규약을 알아도 초안 단계에서는 영어 골격에 한국어 단어를 얹은 문장이 섞이므로,
   이 패스는 생략하지 않는다. 걷어낼 것:
   - 영어 은유 직역 — "~의 함수", "~의 영토", "납작해졌다", "풍경" 류.
   - 문두 부사 + 쉼표 — "역설적이게도," / "흥미롭게도," → 문장 안에서 풀어 쓴다.
   - 쉼표 삽입절 — "정답이 있는 일은, 아무리 고급이어도, ~" → 한 호흡으로 다시 쓴다.
   - 무생물 주어 + 타동사 — "이 기능이 ~를 가능하게 합니다" → 사람/사물이 하는 일로.
   - 판별법: 소리 내어 읽었을 때 한국 개발자가 옆 사람에게 하는 말로 자연스러운가.
7. **파일 작성** — `drafts/<slug>.md` (프런트매터 + 본문).
   - **슬러그는 ASCII 영문만**(`a-z 0-9 -`), 주제 키워드 3~6단어. `slug:` 반드시 명시.
   - `category: insights` 기본. `source_url` 은 쓰지 않는다(원문 없음).
8. **적재** — `npm run draft -- push drafts/<slug>.md` 실행.
9. 출력된 `/admin/editor?slug=…` 경로를 사용자에게 알린다. 검토 후 에디터에서 발행한다.

장르 규약 상세: `INSIGHT.md` · 파이프라인 상세: `docs/draft-in-claude-code.md`.
